# ВХОДНЫЕ ДАННЫЕ (Концепция AFL)

## 1. Базовые сущности AFL

```
- agent: Роль, модель, инструменты, ограничения (guardrails)
- artifact: Общие ресурсы (документы, репозитории, билды)
- mission: Задачи с критериями приёмки
- workflow: Пайплайн взаимодействия агентов
- guardrail: Правила безопасности и качества
- communication: Протоколы передачи между агентами
```

## 2. Требования к оркестратору

- Парсинг декларативных конфигов (YAML-подобный синтаксис)
- Машина состояний для управления workflow
- Context Manager для сжатия и передачи контекста между агентами
- Guardrail Engine для проверки выводов агентов
- Budget Tracker для учёта токенов и затрат
- Интеграция с LLM-провайдерами (OpenAI, Anthropic, etc.)
- Интеграция с внешними инструментами (Git, Jira, Unity, etc.)
- Хранение состояния артефактов и истории чатов
- Асинхронное выполнение задач агентов

## 3. Целевая архитектура (MVP)

```
/orchestrator
  /core
    parser.py       # Читает YAML-конфиги
    engine.py       # Машина состояний workflow
    guardrails.py   # Проверки безопасности
  /agents
    [agent_type].py # Логика конкретных агентов
  /config
    project.yaml    # AFL-конфиг проекта
  /storage
    artifacts/      # Хранение артефактов
    logs/           # Логи выполнения
  main.py
```

   сбоев (перезапуск workflow с точки отказа).
